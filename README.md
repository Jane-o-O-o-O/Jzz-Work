分页查询： ：当数据库中的数据记录很多时，为了增加页面加载速度，提升用户体验，将记录分成很多页，用户通过分页导航读取显示需要的某页数据。每页显示的数据也可以自由选择。

1. CRUD 基本概念

CRUD 代表四类最基础的数据操作：

C – Create（增加）

R – Retrieve（读取/查询）

U – Update（更新）

D – Delete（删除）

在本课中，所有 CRUD 操作均通过 AJAX 异步方式 完成，因此无须刷新整个页面，用户体验更接近 C/S 程序的顺滑操作。

2. 功能需求分析
2.1 查询功能

查询模块需要实现以下能力：

组合查询：支持多个字段条件自由组合，且部分字段支持模糊查询
（如姓名模糊匹配）。

分页查询：大数据量场景下分页返回，提升加载速度。用户可选择每页条数。

排序输出：允许按任意字段升序或降序排序。

结果显示于表格：所有查询结果以表格形式呈现。

表格交互优化

为了提升可读性和体验，需要：

隔行换色：奇偶行不同背景色。

行高亮：鼠标悬浮行高亮显示。

批量选择：第一列提供复选框，用于批量删除等操作。被选中行需高亮。

2.2 删除功能

删除包括两类：

单条删除：点击某行的删除按钮，即发送 AJAX 请求删除该条记录。

批量删除：选中多条记录，提交后端批量删除接口。

删除成功后无需刷新页面，只更新当前表格数据。

2.3 修改功能

修改过程包括：

点击某行“编辑”

行内输入框打开，可修改对应字段

点击“保存”后使用 AJAX 将更改提交到后端

成功后更新该行显示

无需跳转页面或整体刷新。

2.4 增加功能（Create）

新增数据流程：

在页面顶部或弹窗中填写新增表单

点击“新增”按钮

使用 AJAX 将数据以 JSON 或表单形式提交后端

后端新增成功 → 页面追加新行或刷新列表

3. 系统设计

系统设计主要包括 页面设计、交互设计、控制器设计、DAO设计 四大块。

3.1 页面设计（前端）

核心页面元素包括：

查询输入框及组合查询区域

新增/批量删除按钮区域

分页导航

排序按钮（如表头点击排序）

列表展示表格

行内编辑输入框

高亮、隔行换色等 CSS 配置

页面整体布局兼顾易操作性与美观性。

3.2 交互设计
3.2.1 人机交互（前端 behaviors）

主要交互：

输入框输入 / 改变 → 自动更新搜索条件

鼠标悬浮行 → 行高亮

勾选复选框 → 行样式变化

点击排序按钮 → 重新请求数据

点击分页按钮 → 加载对应页数据（AJAX）

点击编辑 → 行内编辑模式开启

所有交互均为前端动态渲染，无需页面跳转。

3.2.2 客户端与服务器端交互（前后端 API）

后端主要提供如下接口（动作以 AJAX 方式调用）：

功能	请求方法	说明
查询	GET/POST	返回分页、排序后的数据列表
新增	POST	新增记录
更新	POST/PUT	修改某条记录
删除	POST/DELETE	删除单条或多条记录

客户端任务：

发送 AJAX 请求（XMLHttpRequest / jQuery AJAX）

接收 JSON 响应

根据响应结果更新页面 DOM

3.3 控制器层设计（Controller）

控制器实现后端业务逻辑路由，例如：

/query.do

/add.do

/update.do

/delete.do

/deleteBatch.do

控制器负责：

接收前端参数

调用 Service

返回 JSON 格式的数据（例如分页结果、成功/失败信息）

3.4 DAO 层设计

DAO 层主要通过数据库操作实现 CRUD，如：

selectByCondition

insert

updateById

deleteById

deleteBatch

保证数据的持久层操作稳定、安全。
